# Contributing to SafeGuard EWS

First off, thank you for considering contributing to SafeGuard! It's people like you that make SafeGuard such a great tool for psychosocial health surveillance.

## 🌈 Code of Conduct

By participating in this project, you agree to abide by our Code of Conduct. We expect all contributors to maintain a professional and respectful environment.

## 🐛 How to Report Bugs

If you find a bug, please create an issue on GitHub with the following information:
- A clear and descriptive title.
- Steps to reproduce the bug.
- Expected vs. actual behavior.
- Screenshots (if applicable).
- Your environment (browser, OS, device).

## 💡 Feature Requests

We welcome new ideas! To suggest a feature:
- Check if the feature has already been requested.
- Open an issue with the tag `enhancement`.
- Describe the problem the feature solves and how it should work.

## 🛠️ Pull Request Process

1. **Fork the repo** and create your branch from `main`.
2. **Install dependencies** using `npm install`.
3. **Make your changes**. Ensure your code follows our coding standards.
4. **Run linting** using `npm run lint` to ensure no syntax errors.
5. **Update documentation** if you've added new features or changed existing ones.
6. **Submit a Pull Request** with a clear description of your changes.

## 🎨 Coding Standards

- **TypeScript**: Use strict typing. Avoid `any` whenever possible.
- **Tailwind CSS**: Use utility classes for styling. Follow the mobile-first approach.
- **Components**: Use functional components and hooks. Keep components small and reusable.
- **Icons**: Use `lucide-react` for all icons.
- **Animations**: Use `motion` (motion/react) for transitions and micro-interactions.

## 🔒 Security & Privacy

Since SafeGuard handles sensitive health data, please:
- Never commit API keys or secrets.
- Ensure all Firestore queries follow the security rules.
- Respect user privacy and data anonymization principles.

Thank you for contributing to a **High-Value Health System**!
